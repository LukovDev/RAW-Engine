### Краткое описание:
Программный движок. Ядро написано на **C**. Обёртка и основная часть на **C++**. Возможно использование **Python** как модули и модификации (_планируется_).</br>
**_Основная суть:_** Создание виртуальной искусственной, или "натуральной" симулированной среды, и наблюдение/исследование виртуального искусственного мира.</br>
_Проект является одним из самых масштабных работ разработчика **LukovDev** (на момент 2026 года)._</br>

#

### Зачем нужен RAWE?
Этот проект должен дать начало направлению разработки своих полноценных симуляций миров, и помочь исследованию и улучшению построения искусственных миров в компьютерных вычислительных системах.</br>

Это не тоже самое, что **"VR/AR"** _(и подобное)._ Это именно **полноценное**, технически возможное, симулирование миров.</br>
Это несовсем "игра", где всё строится вокруг игрока и оптимизациях. Это **симуляция среды и её исследование.**</br>

Проще представить проект как игра "God Sandbox" (или подобное) но под масштабом **NASA**.

### Технические детали:

Графическая составляющая сейчас разрабатывается на `SDL3` и `OpenGL` _(3.3+)_.</br>
Используется `GCC` компилятор. Стандарты: `c17` и `c++17`</br>

#### Сборка и запуск:
Проект основан на моём фреймворке [CPF](https://github.com/LukovDev/C-Program-Framework), который имеет свою систему сборки на пайтон.</br>
Для более подробной информации о том, как она работает и как настроить/использовать систему сборки, можно узнать на странице фреймворка.</br>

> **Для сборки проекта**:</br>

Используйте `build.bat/.sh` файл, который должен собрать весь проект. В случае ошибки или проблемы, сборка будет остановлена.</br>

> **Для запуска проекта**:</br>

Используйте `run.bat/.sh` файл, который должен запустить собранную программу. Запускайте программу из корневого каталога проекта!</br>

> **Использую VS Code**:</br>

Для этого можно использовать `debugpy` и конфигурацию запуска из файла `.vscode/launch.json`. Я просто использую комбинацию клавиш `CTRL+B` (у вас может быть другая комбинация для запуска сборки).</br>

Вы можете использовать другой подход: Используйте примерно такую команду для запуска скрипта, который запускает сборку и запуск собранной программы подряд:</br>
`"${workspaceFolder}/build/tools/all.py"` - Вы можете использовать её в своих конфигурациях запуска в других IDE, главное подредактируйте под себя, сохранив путь запуска Python скрипта `$project/build/tools/all.py`

> **Как сделать компактную сборку для распространения**:</br>

Допустим вы решили сделать свою версию программы, и сделать готовую компактную сборку чтобы поделиться с другими.</br>
Для этого скопируйте всё что находится в каталоге `build/out/*` в другой ваш каталог, а также скопируйте папку `data/` из корневого каталога проекта.</br>

Пример: ProgramFolder/
- `engine.exe` - Сама программа.
- `data/` - Папка данных.
- `...` - Прочие данные, необходимые для работы программы (дин. библиотеки и другое).

### Статус проекта:

Проект поделён на 4 этапа разработки:
1. Разработать движок для симулятора - Разрабатывается (актуально, утверждено).
2. Разработать симулятор миров - Следующее. Ожидание (утверждено).
3. Массовая разработка ИИ для симулятора - В планах (не утверждено. Возможны корректировки).
4. Разработка гарнитур управления и погружения - В планах (не утверждено. Возможны корректировки).

Дорожная карта разработки движка до следующей стадии разработки (движок -> симулятор):

0. [x] Разработать фундамент движка и базу для работы с графикой.
1. [x] Работа с моделями.
2. [ ] Работа ресурсного менеджера - В процессе.
3. [ ] Загрузка OBJ файлов - В процессе.
4. [ ] Внедрение Assimp библиотеки.
5. [ ] Рендерпайплайн для 3D.
6. [ ] Источники света.
7. [ ] Работа с тенями.
8. [ ] Работа с каскадами теней.
9. [ ] Отбрасывание невидимых объектов.
10. [ ] PBR и постпроцессинг.
11. [ ] Простая система частиц.
12. [ ] Поддержка вывода текста и спрайтов.
13. [ ] Базовая поддержка звука.
14. [ ] Поддержка ECS.
15. [ ] Перенос поддержки 2D из PyGDF.

После реализации всех вышеперечисленных пунктов, будет релиз ранней версии движка и переход на следующий (основной) этап разработки проекта (разработка симулятора).
